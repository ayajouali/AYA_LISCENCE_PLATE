OVERVIEW
--------
This dataset contains synthetically generated license plate images
designed for motion blur deconvolution research.
All images are generated programmatically using OpenCV — no real
license plate photos are used (no privacy concerns).

DATASET STATISTICS
------------------
Total samples       : 800
Training split      : 700 samples (indices 0–699)
Validation split    : 100 samples (indices 700–799)
Image size          : 420 x 140 pixels (width x height)
Color space         : BGR (OpenCV default), 3 channels
File format         : Stored as NumPy arrays in dataset.npz

BLUR PARAMETERS
---------------
Blur type           : Linear motion blur (one direction)
Blur length range   : 5 – 20 pixels
Blur angle range    : 0° – 175°  (0° = horizontal, 90° = vertical)
Noise added         : Gaussian noise, std = 0.003 (very light)

Each sample in the dataset contains:
  - sharp    : the original clear license plate image (ground truth)
  - blurred  : the motion-blurred version (model input)
  - kernel   : the PSF (Point Spread Function) used to create the blur
  - length   : blur length in pixels (integer, 5–20)
  - angle    : blur angle in degrees (float, 0–175)
  - text     : the license plate text string (e.g. "AB-123-CD")

HOW DATA WAS GENERATED
-----------------------
Step 1 — Plate generation:
  A white rectangular plate (240 gray background) is created using
  OpenCV. A random 8-character text in the format "LL-DDD-LL"
  (L=letter, D=digit) is rendered using cv2.FONT_HERSHEY_DUPLEX
  at font scale 2.4 with anti-aliasing.

Step 2 — PSF creation:
  A linear motion blur kernel is generated using bilinear splatting
  along a line at the specified angle and length. The kernel is
  normalized so all values sum to 1 (energy-preserving blur).

Step 3 — Blur application:
  The plate image is convolved with the PSF using FFT-based
  convolution (scipy.signal.fftconvolve) for efficiency.
  Light Gaussian noise (std=0.003) is added to simulate sensor noise.

Step 4 — Saving:
  All 800 samples are saved to a single compressed NumPy file
  (dataset.npz) for fast loading.
